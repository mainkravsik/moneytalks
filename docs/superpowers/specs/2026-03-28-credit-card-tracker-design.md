# Credit Card Tracker — Design Spec

## Цель

Полноценный трекер кредитной карты (СберКарта) с учётом трат, грейс-периодов, процентов и калькулятором погашения. Траты по кредитке отдельные от бюджета.

## Механика СберКарты

- Льготный период: 1 месяц на покупки + 3 месяца на погашение (120 дней)
- Начинается с 1-го числа каждого месяца
- Покупки за январь → погасить до 30 апреля
- Переводы/снятие наличных — без льготного периода, проценты с первого дня
- Мин. платёж: max(долг × 3%, 150 ₽) + начисленные проценты
- Ставка: задаётся при создании карты (например 25.4% годовых)

## Модель данных

### Таблица `card_charges`

```
id              SERIAL PRIMARY KEY
loan_id         FK → loans(id)
amount          DECIMAL(12,2) NOT NULL
description     VARCHAR NOT NULL
charge_type     VARCHAR NOT NULL  -- 'purchase' | 'transfer' | 'cash'
charge_date     DATE NOT NULL
grace_deadline  DATE NULL  -- NULL для transfer/cash (проценты сразу)
is_paid         BOOLEAN DEFAULT FALSE
created_at      TIMESTAMP DEFAULT NOW()
```

**grace_deadline** вычисляется при создании:
- `purchase`: последний день месяца (charge_date.month + 3). Пример: покупка 15 января → 30 апреля
- `transfer` / `cash`: NULL — проценты начисляются сразу

### Изменения в `loans`

Убираем из модели/схем:
- `grace_days` — больше не нужен (грейс вычисляется per-charge)
- `min_payment` — вычисляется как max(долг × 3%, 150) + проценты

Добавляем:
- `grace_period_months: int = 3` — сколько месяцев на погашение после месяца покупки (по умолчанию 3 для Сбера)
- `min_payment_pct: Decimal = 0.03` — процент для расчёта мин. платежа
- `min_payment_floor: Decimal = 150` — минимальная сумма мин. платежа

## API

### Траты по карте

- `POST /api/loans/{id}/charges` — добавить трату
  - Body: `{ amount, description, charge_type, charge_date }`
  - grace_deadline вычисляется на сервере
  - remaining_amount карты увеличивается на amount

- `GET /api/loans/{id}/charges` — список трат
  - Query: `?month=2026-03` (фильтр по месяцу charge_date)
  - Возвращает траты с grace_deadline и статусом (in_grace / overdue / paid)

- `DELETE /api/loans/{id}/charges/{charge_id}` — удалить трату
  - remaining_amount уменьшается на amount

### Статистика карты

- `GET /api/loans/{id}/card-summary` — сводка по карте
  - Общий долг
  - Разбивка по грейс-периодам (группировка по grace_deadline): дедлайн → сумма
  - Долг без грейса (transfer/cash) с начисленными процентами
  - Начисленные проценты за текущий месяц
  - Рассчитанный мин. платёж

### Калькулятор погашения карты

- `GET /api/loans/{id}/card-payoff?monthly_payment=X` — прогноз погашения
  - Возвращает помесячную таблицу:
    ```json
    {
      "months": [
        {
          "month": "2026-04",
          "debt_start": 338712.06,
          "payment": 30000,
          "interest": 1200.50,
          "debt_end": 309912.56
        }
      ],
      "total_months": 14,
      "total_interest": 18500.00,
      "total_paid": 357212.06,
      "recommendations": {
        "zero_interest": 115000,       // платить чтобы не платить проценты вообще
        "close_in_6": 62000,
        "close_in_12": 33000
      }
    }
    ```

### Платёж по карте (изменение существующего)

`POST /api/loans/{id}/payment` — при записи платежа:
- Сначала гасятся просроченные грейс-периоды (самые старые первыми)
- Затем transfer/cash долг
- Затем текущие грейс-периоды (ближайший дедлайн первый)
- Помечает charges как `is_paid` когда полностью погашены
- Обновляет remaining_amount

```

```

### Детальная страница карты (по тапу на карточку)

**Табы:** Траты | Калькулятор

**Таб "Траты":**
- Фильтр по месяцу (← Март 2026 →)
- Список трат: дата, описание, сумма, тип, статус грейса
- Цветовая индикация: зелёный = в грейсе, красный = просрочен, серый = оплачен

**Таб "Калькулятор":**
- Слайдер: "Готов платить X ₽/мес"
- Рекомендации (3 карточки):
  - "Без процентов: 115 000 ₽/мес"
  - "За 6 мес: 62 000 ₽/мес"
  - "За 12 мес: 33 000 ₽/мес"
- Таблица помесячно: месяц | долг | платёж | проценты | остаток
- Итого: срок, переплата

## Что НЕ меняется

- Обычные кредиты (loan_type=loan) — без изменений
- Бюджет — траты по карте не попадают
- Существующий payoff калькулятор для обычных кредитов
- Бот-команды
